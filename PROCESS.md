# 타임 박싱 프로세스 (with Claude Code + Notion)

## 개요
- 15분 단위 타임 블록으로 하루 업무를 관리
- Claude Code와 대화하며 **Dump → Select → Time Box → Execute → Review** 루틴 수행
- 모든 기록은 Notion에 자동 연동

---

## 1단계: 아침 계획 (Morning Planning)

**시작 키워드**: "오늘 계획 세우자"

### Phase 0: Cleanup (전날 정리)
1. `cleanup_previous_day()` → 전날 잔여 데이터 자동 정리
   - Do Today 체크된 업무의 배정 시간 / check-in / check-out 초기화
   - 모든 Do Today 해제 (오늘 새로 선별)

### Phase 1: Dump (브레인 덤프)
1. 사용자에게 오늘 떠오르는 할 일 전부 말해달라고 요청
2. `get_backlog_tasks()` → 기존 업무 TODO 백로그도 함께 보여줌
3. 새로 나온 업무 → `add_task_to_todo()` → 업무 TODO DB에 추가

### Phase 1.5: Validate & Decompose (검증 & 분해)
4. Claude가 각 업무를 검증:
   - 15분 블록 단위로 처리 가능한 크기인지 판단
   - 너무 크면 → 하위 작업으로 분해 제안 (예: "API 설계" → "[API 설계] 엔드포인트 정의 30분 + [API 설계] 스키마 설계 30분")
   - 너무 모호하면 → 구체적 행동으로 명확화 제안 (예: "코드 리뷰" → "PR #123 리뷰 15분")
   - 각 업무별 예상 블록 수(15분 단위) 산정
   - **분해 시 이름 규칙**: `[원래 업무] 하위 작업` 형태로 관계 표현
5. 분해가 필요한 항목은 사용자 확인 후 업무 TODO에 하위 업무로 추가

### Phase 2: Select (선별)
6. 사용자에게 업무 시작 시간, 퇴근 시간, 고정 일정(회의, 점심 등) 확인
7. 가용 블록 자동 계산
   - 예: 09:00~18:00, 점심 12:00~13:00 → (9시간 - 1시간) × 4 = 32블록
8. 검증된 전체 목록 + 예상 소요 블록 표시
9. 사용자가 오늘 할 업무 선택 (가용 블록 초과 시 경고)
10. 선택된 업무 → `set_do_today(id, true)` → "Do Today" 체크

### Phase 3: Time Box (배치)
11. 선별된 업무를 시작 시간부터 순서대로 시간 슬롯에 배치
    - 우선순위 기반 배치 (중요한 일을 에너지 높은 시간대에)
    - 비슷한 업무끼리 묶어서 배치 (배치 처리)
    - 고정 일정(회의, 점심)은 건너뛰고 다음 가용 시간에 배치
    - 예시:
      ```
      API 설계 (6블록)     → 09:00~10:30
      코드 리뷰 (4블록)     → 10:30~11:30
      (점심 12:00~13:00)
      스키마 설계 (4블록)   → 13:00~14:00
      잔여: 18블록
      ```
12. 사용자가 Notion에서 타임박싱 템플릿으로 오늘 페이지 생성
13. `setup_timebox_page()` → 템플릿 페이지를 찾아서 속성 채움 (날짜, 총 블록, 핵심 업무 등)
    - `task_assignments` 전달 → 각 업무별 `set_assigned_time()` 호출하여 배정 시간(date range) 기록
    - 링크드 뷰에서 "배정 시간" 컬럼으로 오늘 일정 확인 가능
14. Notion 페이지 URL 반환

---

## 2단계: 실행 중 (Execution)

### 업무 시작/종료 자동 기록

**키워드**: "~~ 시작할게" / "~~ 끝냈어"

- **"~~ 시작할게"** → 해당 업무 상태를 `In Progress`로 변경 + `check-in` 필드에 현재 시각 기록
- **"~~ 끝냈어"** → 해당 업무 상태를 `Done`으로 변경 + `check-out` 필드에 현재 시각 기록 + 타임박싱 페이지에 체크인 로그 추가

### 중간 점검 (Check-in)

**키워드**: "체크인"

사용 시점:
- 예상보다 일이 빨리/늦게 끝났을 때
- 계획을 조정하고 싶을 때
- 전체 진행상황을 확인하고 싶을 때

동작:
1. 사용자에게 진행상황 확인
2. 타임박싱 페이지에 체크인 로그 추가
3. 남은 시간 재배분 제안 (대화로)
4. 진행률 표시

---

## 3단계: 마감 리뷰 (End-of-Day Review)

**키워드**: "오늘 리뷰"

동작:
1. 사용자에게 한줄 회고 요청
2. `finalize_timebox(page_id, user_review, date_str)` → 리뷰 자동 생성
   - TODO DB에서 Do Today 업무 조회
   - 각 업무별 배정 시간 / check-in / check-out에서 블록 수 계산
   - 리뷰 섹션에 자동 작성: 오늘 요약 → 업무별 실행 → 미완료 업무 → 한줄 회고
   - `append_execution_log()` → 실행 이력 누적
   - `clear_assigned_time()` → 배정 시간 + check-in + check-out 초기화
   - 타임박싱 페이지 속성 업데이트 (총 블록, 완료 블록, 달성률, 상태=완료)
3. 미완료 업무 → `set_do_today(id, false)` 해제 (내일 다시 선별 대상)
4. 미완료 업무 안내 및 이월/삭제 판단 지원

---

## 4단계: 주간 회고 (Weekly Review)

**키워드**: "주간 회고" (매주 금요일 또는 수동)

동작:
1. `create_weekly_review()` → 주간 타임박싱 통계 자동 집계
2. 주간 회고 DB에 페이지 생성 (평균 달성률, 총 블록, 성과/개선점)
3. 업무별 실행 이력 수집 → "📋 업무별 실행 이력" 섹션 자동 생성
4. 사용자에게 안내: "Linked View의 날짜 필터를 이번 주차({시작일}~{종료일})로 수동 변경해주세요"

## Notion 연동 구조

```
📋 업무 페이지
├── 업무 TODO DB     ← 읽기 + "Do Today" 업데이트
├── ⏰ 타임박싱 DB    ← 일일 타임박싱 기록 (자동 생성)
├── 📊 주간 회고 DB   ← 주간 통계 (자동 생성)
├── Daily 회고 DB    ← 기존 (수정 안 함)
├── wiki DB
└── 회의록 DB
```

## 업무 TODO DB 속성

| 속성명 | 타입 | 용도 |
|--------|------|------|
| 이름 | title | 업무명 |
| 상태 | status | 진행 상태 |
| Priority | select | 우선순위 |
| 이슈유형 | select | 이슈 분류 |
| 분류 | multi_select | 대분류(`[업무] TEM`, `[개인]`) + 소분류(동적 추가) |
| Do Today | checkbox | 오늘 할 업무 선택 |
| 배정 시간 | date (range) | 오늘 배치된 시간 슬롯 (start~end), Review에서 초기화 |
| check-in | date | 실제 시작 시각 (Notion 버튼으로 now() 기록) |
| check-out | date | 실제 종료 시각 (Notion 버튼으로 now() 기록) |
| 실행 이력 | rich_text | 누적 기록 (예: "3/5(4블록), 3/6(2블록)"), finalize에서 check-in/out 기반 계산 |

### 데일리 관리 뷰
- **필터**: Do Today = checked AND 상태 ≠ 완료
- **보여지는 컬럼**: 이름, 상태, 배정 시간, check-in, check-out
- Priority는 속성 유지하되 뷰에서는 숨김

### 분류 체계
- 대분류: `[업무] TEM`, `[개인]`
- 소분류: Dump 시 필요에 따라 동적 추가 (예: 개발, 기획, 커뮤니케이션 등)
- 기존 옵션(OF21, OF7, Common)은 과거 데이터 보존을 위해 유지

## 스크립트 사용법

```bash
# API 연결 테스트
python3 time/scripts/notion_client.py test

# DB 초기 생성 (최초 1회)
python3 time/scripts/notion_client.py setup

# 오늘 할 일 조회
python3 time/scripts/notion_client.py today-tasks

# 백로그 전체 조회
python3 time/scripts/notion_client.py backlog
```
